# ЕИС «Контроль поручений»
### Единая информационная система контроля исполнения поручений

> **Дипломный проект** — Автоматизация работы администрации района  
> Стек: **Django 6 + React 19 + Vite 8**

---

## 📌 О системе

**ЕИС «Контроль поручений»** — это информационная система, предназначенная для автоматизации учёта, распределения и контроля исполнения обращений граждан в органах местного самоуправления.

Система обеспечивает полный цикл работы с обращением: от публичной подачи заявки гражданином до формирования ответного документа ответственным исполнителем. Все изменения фиксируются в истории, исполнители получают поручения через личный кабинет, а руководство видит сводную аналитику на дашборде.

### Ключевые возможности

- 📥 **Публичная форма подачи** — гражданин подаёт обращение без регистрации
- 🔎 **Проверка статуса** — гражданин отслеживает своё обращение по регистрационному номеру
- 📊 **Дашборд руководителя** — статистика, графики, просроченные и закрытые поручения
- 📋 **Реестр обращений** — таблица со всеми заявками, поиском и фильтрацией
- 👷 **Личный кабинет исполнителя** — просмотр поручений, смена статуса, комментарии
- 📎 **Вложения** — прикрепление файлов к обращению
- 📝 **Генерация документов** — формирование ответа в формате `.docx` (python-docx)
- 🕓 **История изменений** — автоматическая запись каждого действия с указанием автора
- 🔐 **JWT-аутентификация** — безопасный вход для сотрудников, публичный доступ для граждан

## 🚀 Быстрый старт

### Способ 1 — один клик (Windows)

Дважды кликнуть на `start.bat` в корне проекта.  
Скрипт автоматически запустит бэкенд на порту **8000** и фронтенд на порту **5173**, затем откроет браузер.

### Способ 2 — вручную

#### Требования

| Инструмент | Минимальная версия |
|------------|--------------------|
| Python     | 3.10+              |
| Node.js    | 18+                |
| Git        | любая              |

#### 1. Клонировать репозиторий

```bash
git clone https://github.com/valtureso788/-
cd -
```

#### 2. Бэкенд (Django)

```powershell
cd backend
pip install -r requirements.txt
python manage.py migrate
python manage.py seed_data        # загружает тестовые данные и учётные записи
python manage.py runserver 8000
```

> API доступно на **http://localhost:8000/api/**  
> Django Admin — **http://localhost:8000/admin/**

#### 3. Фронтенд (React + Vite) — новый терминал

```powershell
cd frontend
npm install
npm run dev
```

> Приложение доступно на **http://localhost:5173**

## 🔄 Жизненный цикл обращения

```
Гражданин подаёт обращение
        │
        ▼
  [Новое] ──► Оператор назначает исполнителя
        │
        ▼
  [Назначено] ──► Исполнитель берёт в работу
        │
        ▼
  [В работе] ──► при необходимости
        │             [Выезд на место]
        ▼
  [Исполнено] ──► Оператор проверяет и закрывает
        │
        ▼
  [Закрыто]   или   [Отказ]
```

Каждый переход статуса фиксируется в **истории изменений** с указанием пользователя, времени и комментария.

## 📊 Модели данных

### Appeal (Обращение)

| Поле                  | Тип              | Описание                            |
|-----------------------|------------------|-------------------------------------|
| `registration_number` | CharField        | Уникальный номер вида `ГГГГ-XXXX`   |
| `citizen_full_name`   | CharField        | ФИО гражданина                      |
| `citizen_phone`       | CharField        | Контактный телефон                  |
| `citizen_address`     | CharField        | Адрес проживания                    |
| `citizen_email`       | EmailField       | Email (необязательно)               |
| `category`            | TextChoices      | Категория: ЖКХ, Транспорт, и др.   |
| `subject`             | CharField        | Тема обращения                      |
| `text`                | TextField        | Полный текст обращения              |
| `status`              | TextChoices      | Статус: новое → закрыто             |
| `assigned_to`         | FK → User        | Назначенный исполнитель             |
| `deadline`            | DateField        | Срок исполнения (авто: +14 дней)    |
| `resolution_text`     | TextField        | Текст решения/ответа                |
| `created_at`          | DateTimeField    | Дата регистрации                    |
| `registered_by`       | FK → User        | Кто зарегистрировал                 |

### Статусы обращения

| Статус       | Код            | Описание                          |
|--------------|----------------|-----------------------------------|
| Новое        | `new`          | Только что подано                 |
| Назначено    | `assigned`     | Назначен исполнитель              |
| В работе     | `in_progress`  | Исполнитель взял в работу         |
| Выезд        | `on_site`      | Исполнитель выехал на место       |
| Исполнено    | `done`         | Работа выполнена                  |
| Отказ        | `declined`     | Отказ в рассмотрении              |
| Закрыто      | `closed`       | Закрыто оператором                |

## 🔒 Безопасность

- Аутентификация на базе **JWT** (access-токен 8 часов, refresh-токен 7 дней, с ротацией)
- Публичные эндпоинты (подача обращения, проверка статуса) не требуют токена
- Защищённые эндпоинты проверяют роль пользователя на уровне view и сериализатора
- CORS настроен только для разрешённых источников
- В продакшене необходимо заменить `SECRET_KEY`, отключить `DEBUG`, перейти на PostgreSQL

## 🔧 Переменные окружения (продакшен)

При развёртывании на продакшен-сервере рекомендуется вынести чувствительные параметры в `.env`:

```env

DEBUG=False
ALLOWED_HOSTS=yourdomain.ru,www.yourdomain.ru
DATABASE_URL=postgres://user:password@localhost:5432/eis_db
CORS_ALLOWED_ORIGINS=https://yourdomain.ru
```
