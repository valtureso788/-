# ЕИС «Контроль поручений»
### Единая информационная система контроля исполнения поручений
---

## 📌 О системе

**ЕИС «Контроль поручений»** — это информационная система, предназначенная для автоматизации учёта, распределения и контроля исполнения обращений граждан в органах местного самоуправления.

Система обеспечивает полный цикл работы с обращением: от подачи заявления гражданином до формирования и выгрузки официального ответа ответственным исполнителем.

### Ключевые возможности

- 🏠 **Стартовый экран (лендинг)** — центральная точка входа в систему с возможностью быстрого выбора ролей и разделов.
- 📥 **Публичная форма подачи** — гражданин подаёт обращение без регистрации, прикрепляя необходимые документы (до 5 файлов).
- 👤 **Личный кабинет гражданина** — поиск и отслеживание поданных обращений по email и регистрационному номеру, просмотр прогресса и текста ответа.
- 🔎 **Проверка статуса** — быстрая проверка статуса обращения по уникальному номеру.
- 📧 **Email-уведомления** — автоматические оповещения граждан (о регистрации обращения, смене статуса, закрытии) и исполнителей (о назначении поручения).
- 📊 **Дашборд руководителя** — визуализированная сводная статистика и графики статусов обращений.
- 📋 **Реестр обращений** — полнофункциональный реестр обращений с поиском и фильтрацией для операторов и администраторов.
- 📥 **Экспорт в Excel/PDF** — выгрузка реестра обращений в форматы `.xlsx` и `.pdf` с сохранением примененных фильтров.
- 👷 **Личный кабинет исполнителя** — персональный раздел сотрудника для просмотра назначенных поручений, смены статусов и добавления комментариев.
- 🕓 **История изменений** — лог действий по каждому обращению с фиксацией автора и даты.
- 🔐 **JWT-аутентификация** — авторизация сотрудников через Bearer-токены с механизмом ротации.
- 🧪 **Автоматические тесты** — полный пакет тестов (41 тест) для валидации работы базы данных и API.

## 🚀 Быстрый старт

### Способ 1 — запуск в один клик (Windows)

Дважды кликните на файл `start.bat` в корне проекта.  
Скрипт автоматически запустит Django бэкенд на порту **8000** и React фронтенд на порту **5173**, после чего откроет главную страницу в браузере.

### Способ 2 — ручной запуск

#### Требования

| Инструмент | Минимальная версия |
|------------|--------------------|
| Python     | 3.10+              |
| Node.js    | 18+                |
| Git        | любая              |

#### 1. Клонировать репозиторий

```bash
git clone https://github.com/valtureso788/-
cd -
```

#### 2. Запуск бэкенда (Django)

```powershell
cd backend
pip install -r requirements.txt
python manage.py migrate
python manage.py seed_data        # загрузка тестовых пользователей и обращений
python manage.py runserver 8000
```

> API доступно на **http://localhost:8000/api/**  
> Django Admin — **http://localhost:8000/admin/**

#### Запуск тестов бэкенда (pytest)
```powershell
python -m pytest tests/ -v
```

#### 3. Запуск фронтенда (React + Vite) — в новом окне терминала

```powershell
cd frontend
npm install
npm run dev
```

> Приложение доступно на **http://localhost:5173**

| `executor2` | `executor123` | Исполнитель — Транспорт      | Личный кабинет исполнителя          |
| `executor3` | `executor123` | Исполнитель — Соцзащита      | Личный кабинет исполнителя          |

## 🔒 Безопасность

- Аутентификация на базе **JWT** (access-токен 8 часов, refresh-токен 7 дней, с автоматической ротацией)
- Публичные эндпоинты (подача обращения, проверка статуса, поиск обращений гражданина) доступны без токена
- Защищённые эндпоинты проверяют роль и права пользователя на уровне view и сериализатора
- CORS и CSRF настроены только для доверенных доменов
- В продакшене автоматически используется СУБД PostgreSQL на основе `DATABASE_URL` (например, на Render)
